Синхронізувати користувачів автоматично через завдання cron

Увімкніть цей параметр, щоб активувати завдання в cron, яке автоматично створює та / або оновлює облікові записи користувачів на основі записів на сервері LDAP. За замовчуванням, це завдання cron буде виконуватися один раз на день у північ (час сервера). Відредагуйте запис у таблиці "auth_cron" або скористайтеся додатковим сценарієм командного рядка, який надається у htdocs / auth / ldap / cli / sync_users.php, якщо ви бажаєте запланувати його виконання в інший час або з іншими параметрами.

Це налаштування не матиме ефекту, якщо cron не запущено. Див посібник зі встановлення інструкції щодо налаштування.
