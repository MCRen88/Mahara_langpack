Синхронізація груп автоматично через завдання cron

Увімкніть це налаштування, щоб активувати завдання в cron, яке автоматично створюватиме та / або оновлювати групи та їхнє членство на основі записів на сервері LDAP. За замовчуванням, це завдання cron буде виконуватися один раз на день у північ (час сервера). Відредагуйте запис у таблиці "auth_cron" або використовуйте додаткові сценарії командного рядка у htdocs / auth / ldap / cli / якщо ви бажаєте запланувати його для запуску в інший час або з іншими налаштуваннями.

Примітка. Також потрібно активувати налаштування "Синхронізація груп, що зберігаються як об'єкти LDAP" та / або "Синхронізація груп, що зберігаються як атрибути користувача", щоб групи могли бути синхронізовані.

Члени групи можуть бути видалені, а також додані цим параметром. Якщо група більше не знаходиться в записах LDAP, всі її учасники будуть видалені з групи.

Це налаштування не матиме ефекту, якщо cron не запущено. Див Інструкцію з інсталяції щоб дізнатися, як налаштувати її.
